# MVP Overview

This directory contains documentation for the MVP version of the Requests and Offers application.

## Purpose

The MVP delivers a peer-to-peer community platform where users can post, discover, and coordinate exchanges of services and skills. The application evolves from a simple bulletin board into a full conversation-first exchange system across three milestones.

## Core Features (Delivered)

1. **Request/Offer Creation** - Simple forms to create requests and offers
2. **Listing Management** - Users can archive or delete their own listings
3. **Search & Browse** - Public can search and browse active listings
4. **Contact Display** - Clear display of contact information for direct communication
5. **User Dashboard** - Simple dashboard to manage user's own listings

## Upcoming MVP Features

6. **hREA Completion & Real-time Signals** - Complete entity mapping end-to-end, real-time signals, and type safety improvements
7. **Chat System** - Conversation-first messaging with 1-to-1 chats, request/offer linking, and notification system ([#91](https://github.com/happenings-community/requests-and-offers/issues/91))
8. **Exchange Process** - hREA Agreement/Commitment/EconomicEvent lifecycle with conversation-embedded proposals ([#90](https://github.com/happenings-community/requests-and-offers/issues/90))
9. **Reputation System** - Review and rating mechanisms tied to exchange completion

## What's Not Included (Post-MVP)

Features deferred beyond MVP:

- Advanced matching algorithms

These features will be implemented after the MVP based on user feedback.
